# Context Audit Skill — Design

## Overview

`/context-audit` is an OpenCode skill that analyzes session context for dead-weight
tool output — failed commands whose errors were later fixed. All detection and
classification is done by a Python script; the AI simply runs it and presents
the results.

## Architecture

```
/context-audit
    │
    ├── AI runs script:
    │     context_audit.py <session_id>
    │       ├── connects ~/.local/share/opencode/opencode.db (read-only)
    │       ├── Section A: context composition table
    │       ├── Section B: failed tool calls, classified into
    │       │     NOISE (retried → dead weight)
    │       │     INVESTIGATION (not retried → still relevant)
    │       └── outputs the complete report to stdout
    │
    └── AI presents script output verbatim
```

No AI judgment. The script does everything. Classification is mechanical:
`retried = YES` means the failure was resolved, making its output dead weight.

## Script Output

### Section A: Context Composition

| Column | Source |
|--------|--------|
| part type | `json_extract(part.data, '$.type')` |
| count | `COUNT(*)` |
| estimated chars | sum of text length or tool output length per type |
| percentage | type bytes / total bytes |

### Section B: Failed Tool Calls

Detection: `part.state.status == "error"`.

For each failed call, the script checks whether the same command was later
retried and succeeded. If yes → NOISE. If no → INVESTIGATION.

Report format:

```
FAILED TOOL CALLS: 5 found, 3 are dead weight (1.4% of tool output bytes)

  [NOISE] bash   turn 12  328 chars  "cmake --build build"
          Error: undefined reference to 'foo'
          Retried at turn 14 and succeeded → dead weight.

  [NOISE] bash   turn 18  410 chars  "ctest -R failing_test"
          Error: 1/1 tests failed
          Retried at turn 22 and succeeded → dead weight.

  [INVESTIGATION] grep  turn 41  42 chars  "class NotDefined"
          Error: no results found
          Never retried → may still be under investigation.

  [NOISE] bash   turn 8   215 chars  "cd build && cmake .."
          Error: CMakeCache.txt not found
          Retried at turn 9 and succeeded → dead weight.

  [INVESTIGATION] bash  turn 39  156 chars  "npm test -- -t edge-case"
          Error: timeout after 30s
          Never retried → may still be under investigation.
```

Recommendations (generated by the script):

- 3 dead-weight calls are prunable → suggest enabling `compaction.prune`.
- 2 investigation calls are in recent turns → do not suggest removal.

Edge cases handled by the script:

| Scenario | Behavior |
|----------|----------|
| DB missing or unreadable | Print error to stderr |
| Session < 50 parts | "Session too small for meaningful audit" |
| Zero failed tool calls | "Context is clean — no failed tool calls detected" |

## AI Workflow (SKILL.md)

Trivial. The skill tells AI:

1. Run `python3 context_audit.py <session_id>`.
2. Present the output verbatim.

## Non-Goals

- **No automatic cleanup.** The skill is read-only. It does not modify the
  OpenCode database or mark parts as compacted.
- **No persistent report storage.** Output goes to terminal only.
- **No real-time monitoring.** Manual trigger only.
- **No cross-session aggregation.** One session per invocation.
